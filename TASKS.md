## Decisions log

- 2026-05-20 ‚Äî **v9 smoke ladder ‚Üí reward normalization, then full 4M launch.** v9's first 500K smoke ([`ppo_v9_smoke.yaml`](configs/ppo_v9_smoke.yaml), LR 2.5e-4 inherited from v8) caught a value-fitting failure: `explained_variance` pinned at exactly 0 for the full 100K observed, `value_loss` diverging 782‚Üí1525, while the policy itself improved (eval 34‚Üí126). Diagnosis: the deeper IMPALA net + larger return targets from `action_repeat=8` overwhelm v8's LR. **Fix 1 ‚Äî LR 2.5e-4 ‚Üí 1e-4** ([`ppo_v9_smoke_lr.yaml`](configs/ppo_v9_smoke_lr.yaml), extended to 500K): killed the divergence (EV climbed off zero) but EV then *ceilinged at ~0.4* (vs v8's ~0.7) with `value_loss` still ~500‚Äì1500 and a noisy, non-trending eval curve (best 718 was a fluke; avg ~300). Key insight: EV is scale-invariant, so the residual problem is the **return-target scale/variance** from AR=8's chunky per-step rewards, not the LR ‚Äî lowering LR further would only learn *slower*. **Fix 2 ‚Äî `normalize_reward: true`** (SB3 VecNormalize, `norm_reward` only, `norm_obs=False`; [`ppo_v9_smoke_vecnorm.yaml`](configs/ppo_v9_smoke_vecnorm.yaml)) was decisive: EV 0.4‚Üí**0.9**, `value_loss` ~800‚Üí**~0.1**, eval return @500K ~245‚Üí**2608** on a smooth monotonic climb. Implementation decisions: (1) **VecMonitor sits *inside* VecNormalize** (`VecNormalize(VecMonitor(SubprocVecEnv(...)))`) so `rollout/ep_rew_mean` logs raw reward (comparable across runs) while PPO trains on normalized reward. (2) **`norm_obs=False`** ‚Äî images are scaled in the CNN forward, and the bare eval env (which reports *raw* returns) would otherwise see a different obs scale; keeping obs raw means train/eval obs distributions match exactly. (3) **`gamma` is threaded from `cfg.ppo.gamma`** ‚Äî the return-variance estimate is gamma-dependent; a mismatch silently corrupts the scale. (4) **Resume-safe stats**: VecNormalize running averages are saved as a `.pkl` sidecar next to each `step-<N>.zip`/`best.zip` (atomic write, pruned in lockstep) and reloaded via `VecNormalize.load` on `resume_from`; without this a crash-resume would reset the reward scale to fresh. Eval/inference never need the stats (norm_obs=False, no reward norm at inference). Config hygiene: `ppo_v9.yaml` now bakes in LR 1e-4 + `normalize_reward: true`; `ppo_v9_smoke_lr.yaml` pinned back to `normalize_reward: false` so it stays a faithful record of the LR-only rung. 3 new checkpoint tests (pkl written for step+best, no-pkl-without-vecnorm, pkl pruned with zip); the `_FakeModel` stub gained `get_vec_normalize_env`. Full 4M run launched (PID 36682, ~21‚Äì22 h). One pre-existing unrelated test failure remains (`test_load_env_config_from_repo_yaml`: stale `survival_bonus` 0.01 assertion vs env.yaml's v8-retuned 0.03).
